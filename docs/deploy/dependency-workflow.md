# Dependency Update Workflow

Use this required sequence any time dependencies are changed.

1. **Update dependencies in `package.json`**
   - Add, remove, or change versions only in `package.json`.
2. **Regenerate lockfile in the pinned Node/npm environment**
   - Use the project's pinned Node/npm toolchain and run npm so `package-lock.json` is generated by npm.
3. **Run `npm run build`**
   - Confirm the project builds successfully with the updated dependency graph.
4. **Commit `package.json` and `package-lock.json` together**
   - Keep manifest and lockfile in the same commit to preserve deterministic installs.

## Lockfile policy

- Do **not** edit `package-lock.json` by hand.
- `package-lock.json` must only be produced by npm commands.
